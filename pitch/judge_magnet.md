# Judge Magnet

## Personal/customer scar

Every judge who has used AI has lost context in a long chat or needed to hand messy work to another person. Memory Will turns that familiar scar into a visible first action: kill the agent and see whether the work survives without hidden local state.

## 2-3 P0 product guarantees

- P0-1: Durable Memory Will creation after predecessor death.
- P0-2: Cold successor restore from scoped memory only.
- P0-3: Inspectable Memory Will Receipt with honest recovery states.

## Attention ladder

- 0-10s: live predecessor workbench, checkpoint-ready chip, one dominant `Kill Agent` action.
- 10-30s: judge kills agent and chooses successor role/scope.
- 30-60s: Memory Will writes to Walrus/MemWal, cold successor opens, inherited/unavailable memory appears.
- 2-3min: judge mutates role or redaction and sees output/proof change.
- 5min: judge reopens receipt and inspect surface from a fresh context.

## Judge success metrics

- First action happens in <=30 seconds.
- 5-second clip is understandable without builder narration.
- Successor output changes when role or redaction changes.
- Receipt can be reopened and inspected by pointer.
- Missing durable memory blocks restore instead of showing fake success.

## Ship cut

Ship only the guided research-comparison beachhead, scoped Memory Will write/read/restore, role/redaction mutation, receipt, inspect surface, and one honest degraded state. Cut general agent framework, enterprise identity, admin dashboards, and broad workflow templates.
