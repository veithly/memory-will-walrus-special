# User Cases

HERO PATH: kill a predecessor agent, write a scoped Memory Will, and restore a cold successor from durable memory.

## User case 1 - Hero path: research succession

- User: research/product lead comparing vendors or options with an AI agent.
- Trigger: the original agent, browser tab, contractor, or tool cannot continue.
- Action: user creates a Memory Will, scopes what a successor can inherit, and opens a cold successor agent.
- Result: the successor resumes from durable Walrus/MemWal memory with visible inherited facts, redactions, and replay.
- Return reason: future long-running agent work can be handed off, restored, or audited instead of pasted from a chat transcript.

## User case 2 - Sales account handoff

- User: sales manager or account executive.
- Trigger: account owner leaves, switches territories, or hands work to a specialist.
- Action: create a Memory Will containing allowed customer context, objections, next steps, and redacted sensitive notes.
- Result: successor sales agent drafts a role-specific next action while proving which memories were inherited.
- Buyer: sales organizations and CRM/AI workflow platforms.

## User case 3 - Legal matter handoff

- User: legal operations team or small law firm.
- Trigger: one agent or teammate cannot continue review of a matter.
- Action: delegate only approved facts, sources, and open questions to a successor legal-review agent.
- Result: successor continues with explicit redactions and uncertainty where memory is blocked.
- Buyer: law firms, compliance teams, matter-management platforms.

## User case 4 - Research sprint continuation

- User: research lead, analyst, founder, or consulting team.
- Trigger: multi-day research is interrupted or split across roles.
- Action: successor agent inherits a scoped checkpoint and source bundle.
- Result: finance, design, legal, or operator successor produces different next-step analysis from the same durable Memory Will.
- Buyer: consulting firms, accelerators, internal strategy teams, research platforms.

## Non-User Anti-Cases

- A chatbot that summarizes its chat history locally.
- A project handoff page with no cold restore.
- A log viewer where the judge cannot mutate successor role or memory scope.
- A generic workflow engine with Walrus only used to store final reports.
- A scripted two-agent animation where both agents share hidden context.

## Human Approval Boundary

- High-risk actions are not executed automatically.
- The predecessor can write memory and propose a successor scope.
- The user approves what gets inherited.
- The successor may produce draft recommendations and ask for missing memory, but cannot publish external actions without approval.
