## Selected Winner Slot

- Prize / track / rubric behavior: Sui Overflow 2026 - Special - Walrus.
- Required construct, if any: Agent application / functional AI agent workflow using Walrus as durable data and memory infrastructure.
- Required construct primer:
  - What counts: the judge-triggered action writes durable agent memory or an artifact to Walrus, then another agent or workflow reads it back before the result appears.
  - What does not count: optional upload, storage appendix, local-only memory, hidden transcript, or proof page disconnected from agent behavior.
  - Demo proof / first-minute demo proof required: the judge changes scope, Walrus write/read occurs, and the successor output changes from the restored memory.
  - Binding constraint for ideation: Walrus must change the product outcome, not only store a file after success.
- Agent-native loop: the predecessor observes task state, plans a Memory Will, calls tools, mutates durable state, verifies Walrus restore, and replans when the judge blocks memory.

- Durable state objects >=3: predecessor checkpoint, scoped Memory Will, Walrus blob/object metadata, successor replay, and blocked-memory list.
- Surprise replan: if the judge blocks pricing/private notes or Walrus restore lags, the successor continues with unavailable memory named instead of leaking it.
- Replay trace: the receipt timeline shows kill -> write -> restore -> successor output.
