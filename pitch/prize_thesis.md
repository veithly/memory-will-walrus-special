# Prize Thesis: Memory Will

Required before idea generation. One project attacks one primary prize. Backup prizes are allowed only as secondary fit, never as a second story.

## Primary Prize

- Primary prize: Special - Walrus.
- Backup prize: none.
- Why this exact prize is winnable: the Walrus track asks for functional AI agents or workflows that use Walrus as durable data infrastructure for memory, shared context, and portable state. Memory Will makes that primitive the product consequence: a killed agent cannot hand off until the will is written to Walrus and reopened by a cold successor.
- Why we are not optimizing for other prizes: no DeepBook trading, no DeFi payment flow, and no broad agent marketplace story. The public story stays on Walrus-backed agent memory succession.
- Project language: English.
- Language source: DeepSurge event API and public event page use English for Sui Overflow 2026 details and submission surfaces.

## Sponsor Primitive

- Primary sponsor/domain primitive: Walrus Publisher and Aggregator APIs with Sui object metadata for proof inspection.
- User action changed by this primitive: the judge can kill a predecessor agent, write only approved memory, and open a cold successor that reads the durable will before answering.
- What breaks if removed: the successor either inherits hidden local context or reads a pasted summary; scoped restore and inspection collapse.
- Proof surface: `/api/memory-wills`, `/api/successor/restore`, `/api/proof/[willId]`, `.hunter/runtime-interaction.report.json`, and the Memory Will Receipt.

## Target User And Feasibility

- Target user/community/workflow: research, product, sales, and legal teams running long AI work that must survive tool resets or teammate handoffs.
- Buyer / budget owner, if this became a product: AI workflow platforms, consulting firms, research groups, and enterprise teams that need auditable agent memory boundaries.
- Purchase/adoption trigger: agents are moving into longer tasks, but teams still lose context or leak restricted notes when work moves across tools.
- Existing alternative: chat export, shared prompt, project note, vector-store memory, or handoff summary.
- Why our demo is more than a toy: the same engine can handle sales account handoff, legal matter handoff, research sprint continuation, and cross-tool agent workspace recovery.

## 30-Second Demo Moment

- 0-10s hook: a judge sees a live AI task and the `Kill Agent` button.
- 10-30s user action: the judge kills the predecessor, chooses a successor role, and writes allowed memory to Walrus.
- Visible result: local context drops to 0%, a Walrus pointer appears, and the cold successor resumes with only approved memory.
- Proof artifact: Memory Will Receipt with blob id, Sui object id, redactions, replay trace, output hash, and second-context reopen path.

## Killer Artifact

- Proof phrase: "A cold successor replay shows only approved memory."
- Why a judge can repeat it: the first action, state change, and result fit in one sentence.
- Where it appears: README first line, submission one-line description, video first 30 seconds, and the app receipt surface.

## No-Go Trigger

- Kill if: restore succeeds without reading the Walrus blob, or the successor sees hidden local context.
- Pivot if: judges describe the product as a proof dashboard rather than an agent memory succession tool.
- Do not build: a general agent framework, storage badge demo, chat export viewer, or multi-sponsor dashboard.
