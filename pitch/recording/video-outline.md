# Video Outline: Memory Will

## Spine

user -> action -> result -> inspection path

## Beats

1. Hook: a research lead kills a predecessor agent and gets a Memory Will receipt with a Walrus blob id.
2. Pressure: the successor needs approved memory, not private budget notes or unverified hunches.
3. Live action: click `Kill Agent`, write the will, open the cold successor.
4. Result: the successor output changes by role and the receipt lists allowed and unavailable memory.
5. Inspection: click `Inspect Proof`, then reopen `/?will=latest` in a second context.
