## Binding Construct

The judge must cause a Walrus- or MemWal-backed state change in the first minute, then reopen, recall, restore, share, delegate, or inspect the resulting durable object or memory. Walrus cannot be a badge, optional upload, hidden backup, or proof page detached from user action.

## Required Construct Primer

- Required construct: Agent application or functional AI agent workflow using Walrus as durable data and memory infrastructure.
- What counts: the judge-triggered action writes durable memory or an artifact to Walrus, then a successor agent or workflow reads it before producing the result.
- What does not count: optional upload, storage appendix, local-only memory, hidden transcript, static summary, or a proof page disconnected from agent behavior.
- Demo proof: the judge changes successor scope, Walrus write/read occurs, and the cold successor output changes from the restored memory.
- binding prompt constraint: every candidate must make Walrus change the product outcome rather than only store a file after success.

## Agent-Native Fit for Tournament

- Agent-native loop: every advanced candidate must show an agent goal, observe changing state, plan, call tools, mutate durable state, verify the result, and replan when the judge blocks or changes memory.
- Agent tools >=3: checkpoint writer, Walrus publisher/reader, redaction gate, successor generator, and Sui proof inspector are the minimum tool family for Memory Will-shaped candidates.
- Durable state objects >=3: predecessor checkpoint, scoped Memory Will, Walrus blob/object metadata, successor replay, and blocked-memory list.
- Surprise replan: if the judge blocks a memory category or a restore is delayed, the successor must continue with unavailable memory named instead of leaking hidden context.
- Replay trace: the public artifact must show kill -> write -> restore -> successor output.

## Non-Negotiable Bars

- Audience Proximity: a non-domain judge can care and roleplay in the first 5 to 30 seconds.
- Judge Magnet: first action <=30s, 5-second replayable state change, input mutation, share/replay/compare artifact.
- Judge first action <=30s: the judge must personally trigger the first state change.
- 5-second replayable clip: each advanced concept needs a visible before/action/after moment that can be shown again without explanation.
- What changes when judge changes input: role, timing, blocked memory, or user-supplied value must change the result.
- Share/replay/compare artifact: the result must be screenshot-worthy, reopenable, or comparable.
- Nearest boring clone: every candidate names the default boring version it must avoid.
- Mutation that makes it not boring: every candidate names the front-stage twist that makes the clone replayable.
- Agent-Native: visible goal, changing state, tool calls, durable state mutation, verification, surprise/replan, replay trace.
- Product Ambition: one sharp beachhead, at least three adjacent cases, reusable engine, month-1 and month-6 path.
- Walrus Criticality: removing Walrus/MemWal changes the product consequence, not only the storage vendor.

## Judging Slate

The following six candidates are advanced to J1-J3. They cover play/status, money-demand, creator-status, and hard Walrus proof without letting the pool become mostly support, scheduling, or paperwork automation.

### S1: Agent Funeral / Memory Will

- Source ID: I3-2.
- User: research leads, sales teams, founders, project managers.
- First action: judge presses `Kill Agent`, then chooses successor role.
- Replay clip: agent dies, writes a memory will, successor opens scoped Walrus/MemWal memory and resumes.
- Input mutation: successor role or blocked memory category changes delegated state and output.
- Artifact: Memory Will with delegated memory list, redactions, successor replay, compare mode.
- Walrus criticality: successor cannot see local context; only scoped durable memory lets it continue.
- Nearest boring clone: chat export or project handoff summary.
- Mutation that makes it not boring: judge kills the agent and watches a cold successor inherit only scoped memory.
- Product ambition: cross-tool memory succession for long-running agent work.
- Main risk: could be read as theatrical summarization unless restore and delegation are inspectable.

### S2: Avatar Continuity Arena

- Source ID: I1-7.
- User: indie game creators, streamers, tabletop GMs, players.
- First action: judge chooses `Save Dragon` or `Steal Crown`.
- Replay clip: sidekick remembers the choice in a second mini-world.
- Input mutation: moral choice changes relationship, second-world dialogue, quest availability, memory card.
- Artifact: Companion Memory Card with choice, relationship shift, callback, blob id, replay.
- Walrus criticality: cross-world continuity breaks without durable companion memory.
- Nearest boring clone: NPC memory card or scripted game save.
- Mutation that makes it not boring: the judge choice changes behavior in a second world.
- Product ambition: portable memory for AI characters across games, creator worlds, and learning modules.
- Main risk: if too scripted, judges may miss that Walrus is load-bearing.

### S3: BidForge Memory

- Source ID: I2-7.
- User: small contractors, remodelers, landscapers, painters, home-service owners.
- First action: judge drags budget or toggles `Water damage found`, then presses `Rebid`.
- Replay clip: scope and price split, margin warning flashes, job evidence writes, homeowner bid reopens.
- Input mutation: budget/risk changes price, exclusions, margin, assumptions, confidence, bid memory.
- Artifact: homeowner bid card with photos, assumptions, exclusions, price confidence, saved job memory.
- Walrus criticality: durable job evidence, bid assumptions, margin history, and homeowner versions are the product.
- Nearest boring clone: estimating spreadsheet or AI bid generator.
- Mutation that makes it not boring: a budget/risk change visibly rewrites margin memory and homeowner proof.
- Product ambition: bid memory CRM expanding to supplier, financing, insurance, and crew planning networks.
- Main risk: could become back-office estimating unless the money-risk shock is front-stage.

### S4: Memory Tattoo Parlor

- Source ID: I3-1.
- User: creators, brand strategists, small agencies, ecommerce teams.
- First action: judge inks one brand/taste rule onto an agent.
- Replay clip: agent is wiped, reincarnated, recalls the tattoo, and rewrites the artifact correctly.
- Input mutation: tattoo rule changes durable memory, generated artifact, contradiction warnings, replay diff.
- Artifact: Tattoo Card with before/after, tattooed rules, memory pointer, handoff.
- Walrus criticality: blank successor only behaves correctly after recalling Walrus/MemWal memory.
- Nearest boring clone: saved brand settings.
- Mutation that makes it not boring: wipe the agent, tattoo one rule, and show reincarnation obeying it.
- Product ambition: portable taste/brand memory for teams and AI app builders.
- Main risk: could look like personality settings unless the wipe/reopen proof is hard.

### S5: Context Crash Test

- Source ID: I3-6.
- User: developers, research teams, AI-tool builders.
- First action: judge presses `Nuke Context` during a visible task.
- Replay clip: local memory is cleared, Walrus/MemWal restore pulls checkpoint, agent resumes at step 3.
- Input mutation: nuke timing changes restored checkpoint, remaining plan, artifact diff, replay trace.
- Artifact: Crash Receipt with before-nuke state, restored checkpoint, resumed artifact, object pointer.
- Walrus criticality: crash recovery is impossible without durable checkpoint/state objects.
- Nearest boring clone: checkpoint log viewer.
- Mutation that makes it not boring: the judge nukes context mid-task and the resumed artifact proves restore timing.
- Product ambition: portable recovery layer for long-running agent platforms.
- Main risk: can feel infrastructure-shaped if the first beat becomes logs instead of destruction/recovery.

### S6: SponsorTape

- Source ID: I2-2.
- User: creators selling sponsorships directly.
- First action: judge edits a campaign metric or chooses a brand category, then refreshes sponsor kit.
- Replay clip: rate card changes, brand-fit list reshuffles, proof clip anchors, sponsor view reopens.
- Input mutation: sponsor category/metric changes rates, claim confidence, evidence receipts, brand boundaries.
- Artifact: sponsor kit with rate rationale, campaign proof, brand-fit recommendations, delegated sponsor view.
- Walrus criticality: sponsor-safe proof history, brand boundaries, and kit versions need durable delegated memory.
- Nearest boring clone: AI media kit generator.
- Mutation that makes it not boring: sponsor category changes proof, rates, delegated view, and reopenable evidence.
- Product ambition: creator proof rooms for agencies, brand teams, and performance marketplaces.
- Main risk: generic AI creator tool unless Walrus proof/reopen is visible but not a storage lecture.

## Local Red-Team Before Judges

- The pool is valid for J1-J3 because it includes desire-led/playable candidates, money-led demand candidates, and hard Walrus proof candidates. It is not mainly scheduling, support, refund, paperwork, or operator cleanup.
- S1 and S5 have the strongest Walrus primitive but risk becoming infra demos. Judges must test whether the first beat stays human-close.
- S2 has the strongest non-domain magic but needs to prove the second world is not scripted.
- S3 has the strongest buyer but needs a demo that feels like a public money-risk shock, not contractor software.
- S4 has the cleanest visual metaphor and likely easiest polished demo, but must not become local settings.
- S6 has creator money/status and strong artifact value, but must distinguish itself from media-kit generators.
- Any final lock must name at least three durable state objects, at least three tools, surprise/replan behavior, a real inspect path, and a six-month product path.
