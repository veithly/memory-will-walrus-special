Working title: Roast Relay

Seven-word rumor: Your joke survives hecklers and wins receipts.

Specific reachable user: Standup open-mic comics, Twitch streamers, community hosts, Discord mods, and hackathon demo presenters who want sharper audience-tested bits.

Desire / power pull: Look funnier, faster, and more in control than the room. The user is not asking for “AI help”; they want a crowd-tested line that remembers what bombed, what hit, and what the audience tried to derail.

Existing voluntary behavior: People workshop jokes, A/B-test captions, post crowd reactions, save “bits,” remix comments, run live polls, and share clips of heckle comebacks.

Beachhead demo case: A judge chooses a persona, enters a simple premise like “Sui storage is hard to explain,” and lets the room submit one heckle. The agent generates a tight comeback, stores the bit lineage and audience reaction in Walrus/MemWal, then reopens the bit after a forced refresh and adapts based on the remembered heckle.

Judge first action <=30s: Type a premise or pick one, then press Heckle Me.

First 5-second replayable clip: Premise appears → audience heckle lands → agent rewrites punchline → laugh meter moves → “Bit Memory saved” card appears with blob id → refresh → agent recalls “avoid jargon, use the otter joke.”

What changes when judge changes input: Changing the premise changes the bit draft, saved memory object, audience reaction tags, future callback, and replay artifact. Changing the heckle changes the agent’s replan path and the remembered “never again / use again” lesson.

Share / replay / compare artifact: A “Roast Receipt” showing the before line, heckle, comeback, crowd vote, remembered lesson, blob id, and a replay link. It is a flex card, not a proof dashboard.

Walrus or MemWal critical path: The bit is not complete until the bit lineage, reaction tags, and learned callback are written to a Walrus-backed memory space. Reopening the same bit or handing it to a second “closer” agent must use the Walrus/MemWal object. Hidden local history would break the punchline recall.

Agent-native loop: Goal: make the user’s bit win the room. Observe premise + heckle + vote → plan punchline angle → call tools → write durable bit memory → verify audience score → replan if heckle beats the joke → produce replay receipt.

Agent tools >=3: Audience poll/vote collector; style/voice memory recall; joke rewrite tool; Walrus/MemWal remember/recall; replay-card renderer.

Durable state objects >=3: Bit memory space; premise/heckle lineage blob; audience reaction score; callback rule; shareable Roast Receipt.

Surprise / replan path: Judge injects “too mean,” “make it family-friendly,” or “audience boos.” Agent recalls the user’s saved boundaries and rewrites the comeback while preserving the bit history.

What judges can inspect: Blob id for bit lineage; remembered callback; before/after joke versions; agent trace showing recall → rewrite → save; fresh-session replay proving the callback survives refresh.

Product thesis: Durable creative memory for live performers: every audience interaction becomes a reusable, portable style lesson and artifact.

Three adjacent cases: Livestream chat comebacks; debate-club rebuttal practice; sales-demo objection handling.

Expansion ladder: Beachhead: one live joke battle. Month 1: creator workspace for bits, audience tags, callbacks, and persona memory. Month 6: cross-platform performance memory for streamers, presenters, comedians, and community hosts.

Industry buyer behind it: Creator tools, livestream platforms, comedy schools, event platforms, and sales enablement teams.

Nearest boring clone: “AI joke generator with saved history.”

Mutation that keeps it judge-magnetic: Make the room a heckler and make the saved memory visibly improve the next comeback after refresh.

Why it might still lose: It may read too toy-like unless the Walrus-backed handoff and durable creative memory are inspectable and unmistakably load-bearing.

Lane Self-Red-Team

Strongest lane candidate: I1-7 Avatar Continuity Arena has the clearest non-domain first-minute magic: make a choice, cross a portal, and the character remembers. It is also naturally Walrus/MemWal-shaped because cross-world recall, restore, and delegated character memory are the product, not a storage appendix. This is not a final winner pick.

Candidate to kill immediately: I1-2 FitCheck Court is the easiest to collapse into a generic recommendation app. It can be saved only if the crowd challenge and future recalled taste rule are unmistakably durable and inspectable; otherwise Walrus is too removable.

Repeated trap this lane must avoid: Treating Walrus as “where we save the card.” Each candidate must make later behavior change because a durable memory/artifact was reopened, recalled, delegated, restored, or inspected. The R0 red-team explicitly warns that Walrus projects lose when Walrus is removable, when the demo is still a chatbot/memory dashboard, or when portability is claimed without later-session recall, restore, second-client reuse, second-agent reuse, or delegation.
