Candidate Notes
S1 Agent Funeral / Memory Will

Why a tired judge personally cares: Everyone who has used agents or long chats has felt context loss. “Kill Agent” is a violent, funny, instantly understandable button, and successor-role choice gives the judge control within seconds.

Retell line: “You kill an AI, it writes a memory will, and the next AI continues only with the memories you allowed.”

Strongest product wedge: Scoped memory succession for teams with long-running work. The Memory Will artifact, redactions, successor replay, and compare mode make the memory handoff feel owned instead of invisible. The tournament already marks S1 as having very high Walrus criticality, judge magnetism, and agent-native proof.

Boring clone risk: Project summarizer, meeting handoff memo, or “export chat history” tool.

Fatal blocker, if any: Fatal if the successor could continue from local prompt context, pasted transcript, or ordinary summary. It must feel like a scoped durable handoff, not theatrical summarization.

S2 Avatar Continuity Arena

Why a tired judge personally cares: “Save Dragon” versus “Steal Crown” is instantly roleplayable. The magic is not storage; it is that a character remembers your moral choice in another world.

Retell line: “I saved a dragon, and the sidekick remembered it in a different game.”

Strongest product wedge: Portable AI character memory for creator worlds, games, tabletop campaigns, streams, and learning modules. The Companion Memory Card is a replayable artifact with choice, relationship shift, callback, blob id, and replay.

Boring clone risk: Scripted NPC demo, branching story toy, or chatbot-with-lore.

Fatal blocker, if any: Fatal if the second-world behavior is obviously pre-scripted. This candidate lives or dies on whether the judge believes the memory card caused the callback.

S3 BidForge Memory

Why a tired judge personally cares: Home repairs, hidden damage, underbidding, and surprise costs are ordinary-life money pressure. A judge can roleplay the homeowner or contractor without knowing Web3.

Retell line: “When water damage appears, the bid rewrites itself with proof, exclusions, and margin memory.”

Strongest product wedge: This has the clearest buyer and repeat use. Small contractors bid weekly, lose margin on missed assumptions, and can plausibly pay for durable bid evidence and compounding margin memory. The tournament describes S3 as a money-risk shock with durable job evidence, homeowner bid reopen, and expansion into supplier, financing, insurance, and crew planning networks.

Fatal blocker, if any: Fatal if the first minute feels like filling a contractor form. The money-risk shock must be front-stage: budget/risk changes price, exclusions, assumptions, and reopened homeowner artifact.

S4 Memory Tattoo Parlor

Why a tired judge personally cares: People repeatedly tell AI their brand voice, taste, constraints, and “don’t do that again.” Tattooing a rule onto an agent is a sticky metaphor that non-domain judges can understand immediately.

Retell line: “You tattoo a brand rule onto an agent; after it gets wiped, it still writes correctly.”

Strongest product wedge: Portable taste and brand memory for creators, agencies, ecommerce teams, and AI app builders. The wipe/reincarnation moment is judge-magnetic, and the Tattoo Card gives a clean before/after artifact.

Boring clone risk: Personality settings, saved prompts, brand guidelines panel, or style preset manager.

Fatal blocker, if any: Fatal if the tattoo is just local prompt injection. The wipe/reopen proof has to be hard enough that a judge sees memory portability, not settings.

S5 Context Crash Test

Why a tired judge personally cares: The “Nuke Context” button is fun, and most AI users understand losing context. But the buyer/user frame leans developer and AI-tool-builder, so this risks becoming a technical proof before it becomes a product.

Retell line: “It nukes the agent’s memory and restores the exact working state from Walrus.”

Strongest product wedge: Recovery layer for long-running agent platforms. It has strong Walrus criticality because crash recovery is impossible without durable checkpoint/state objects, and the Crash Receipt is an inspectable artifact. 

idea_tournament

Boring clone risk: Checkpoint dashboard, logs viewer, infrastructure demo, or persistence SDK sample.

Fatal blocker, if any: Fatal if the first beat becomes logs. The visible task must be human-legible before the nuke, or only builders will care.

S6 SponsorTape

Why a tired judge personally cares: Creator sponsorship is easy to understand: metrics change, money changes, and the sponsor kit becomes the thing you send to brands.

Retell line: “A creator changes campaign proof, and the sponsor kit rebuilds its rates and brand-fit story.”

Strongest product wedge: Creator money/status artifact with delegated sponsor views. The sponsor kit, rate rationale, proof clips, brand-fit recommendations, and sponsor-safe history are strong on shareability.

Boring clone risk: Generic AI media-kit generator, analytics upload portal, or influencer CRM.

Fatal blocker, if any: Fatal if metrics are just editable fields and the sponsor view is just a generated PDF/card. This one has the highest clone risk because the same front-stage product could exist without Walrus unless proof/reopen is visible but not lecture-heavy.

J2 Recommendation

Advance strongest: S3 BidForge Memory. It has the best hard product case: real buyer, obvious money pain, frequent repeat use, and the clearest beachhead-to-six-month product ambition. It is less magical than S1/S2/S4, but it has the strongest demand spine.

Keep as backup: S4 Memory Tattoo Parlor as the best balanced judge-magnet/product candidate, and S1 Agent Funeral / Memory Will as the strongest Walrus-native handoff candidate. S2 Avatar Continuity Arena is the best pure non-domain magic backup, but buyer/budget is weaker.

Kill: S5 Context Crash Test as a product/audience pick, despite strong Walrus primitive, because it is most likely to become infrastructure-shaped. S6 SponsorTape should be killed as written if it cannot escape the generic media-kit clone.
